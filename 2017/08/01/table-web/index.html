表格（table）设计总结

做了几套后台的网页设计，踩了好多坑，有时被团队质疑，甚至被自己质疑。也参照了ant－design、material design、element－ui，做了一些自己的规范。接下来用markdown总结一部分内容，给自己一个总结，希望可以指正。

一.对齐（align）

左对齐

大部分文字 比如业务编号、名字、公司名称等大部分比较长不确定性的内容。因为它符合了人们大部分的阅读习惯。

居中

长度固定不变的内容 比如时间、身份证号等。其中也有部分人会把改内容归位左对齐

右对齐

数字及小数点之后有固定小数 因为大多数用户习惯“个十百千万的顺序”，方便上下对比数据

二.文字（font）

内容长度

在设置表头的文字要尽量精简，不要让表头的文字占据大量的空间，会严重缩短表格要呈现的内容，尽量控制在4个字左右；表格内部文字也要精简可以适当带着省略号，当鼠标移上去展示全部

字体大小

文字大小尽量控制在12、13px，14px放在表格里偏大

字体样式

文字要简洁干净。不需要衬线字体，不要加粗也不要斜体

三.修饰（decorate）

分割线

在绘制表格时会或多或少加上分割线，分割线的作用：分开数据不一样的信息。1）当我们在用到水平分割线时，会更加强调行，当然我们在权衡要不要使用分割线时会有些纠结。根据格式塔原则，我们要适当把不同的信息进行分类，把同类信息放在一起或更靠近。所以大多数情况下水平分割线都存在，但是线的颜色不能太深，会抢走数据的重要性；2）我们是否要用垂直分割线？如果数据更具有独立性可以加上，这里更强调每列之间的不一样。我个人还是建议去掉垂直分割线，来突出数据本身

每行的背景

这里的背景提及的是斑马纹，斑马纹的作用：让用户在读大量数据时不会感觉疲劳，但是在上下数据对比或者会和鼠标移上去会有些不一样。所以这里还是少用

其他

其他一些大家会觉在视觉上酷炫的效果：投影、3d效果等尽量舍去不要来干扰数据

四.配色（color）

线条的配色：＃eeeeee；

表头的颜色：主色调降低透明度

网站的主色调：对比度要高。根据网页可读指导原则（Web Content Accessibility Guidelines ，WCAG）将对比度分成了三个等级分别为A、AA、AAA。网页中可读文字至少要达到AA级别，即4.5：1，最佳的对比度要达到7：1。

五.状态（status）

我们会遇到这些状态列表：交易中、交易成功、交易失败等，会有很多列表的状态是一样。因为这些内容大量重复，我们很难看出它们的不同，所以需要我们进行可视化（前面加一个状态颜色的小点），让每个状态不一样，并且能够轻易看出它们的不一样。

六.操作（antion）

表内操作

1.查看详情

我们在查看详情操作会有两种情况：第一种是在表格里有一个查看详情的按钮；第二种是点击业务编号的链接进入下一个页面

2.编辑

这里有三种情况：第一种是直接编辑，在点击表格内“编辑”文字来显示输入框的形式；第二种是浮窗编辑，在鼠标移上文字会出现浮窗可以在浮窗进行编辑修改；第三种是弹窗编辑，通过一个修改按钮弹出一个弹窗进行修改。这里个人建议直接修改，会更加直接。

八.分页（pagination）

分页的主要作用是当内容开始多起来，节省一定空间并缓解用户的压力。构成元素：上一页、下一页、页数、跳转到某页（确定按钮）。这里有些组合：1.上一页、数字、下一页 2.上一页、数字、下一页、总共多少页 3.上一页、数字、下一页、总共多少页、跳转到、确定按钮。

问自己几个问题：1）页面数量多少合适？在开始量比较少的时候，表格的分页器尽量限制在1-4页左右。2）查看最多的是哪几个页面？根据大部分的行为习惯说明，用户习惯把点击首页。3）手动翻完页面的概率有多大？4）mouserover的情况怎么显示？可以采用颜色显示。5）用户会不按照页面顺序点击么？ 6）用户会不会翻以前翻过的页面？7）翻页是否应该出现在顶部？什么时候出现在顶部？我们可以看到邮箱和淘宝的宝贝列表会出现，当内容太多，用户想要找的内容在下一页可以不用看内容直接进行跳转。8）是否显示最后一页？在内容太多，用户浏览得不耐烦时，希望尽快完成浏览任务的情况下。
